# 67速通 - 需求拆解文档

## 产品概述

- **产品类型**: 排行榜网站（视频挑战赛平台）
- **场景类型**: <scene_type>prototype-app</scene_type>
- **目标用户**: 短视频挑战赛参与者、速通挑战爱好者、喜欢趣味竞技的年轻用户
- **核心价值**: 让用户上传一分钟内打“67”的挑战视频，AI自动分析67数量并计算分数，按分区展示排行榜，营造游戏速通般的竞技氛围
- **界面语言**: 中文
- **主题偏好**: user_specified（暗红色+亮黄色撞色，游戏速通挑战风格，视觉冲击力强）
- **导航模式**: 锚点导航（单页多区块）

## 页面结构总览

> **说明**：单页应用，通过锚点导航切换区块

**页面文件**: `HomePage.tsx`

| 区块名称 | 锚点 | 区块说明 |
|---------|------|---------|
| 英雄区 | `#hero` | 大标题“67速通”、标语、视觉冲击背景（暗红色+闪电元素+红色熊形象） |
| 视频上传区 | `#upload` | 视频上传按钮、分区选择（3个分区）、视频简介输入、提交按钮 |

## 页面布局建议

- **布局模式**: 上下分区（单栏滚动），每个区块全宽展示，通过锚点导航快速跳转
- **视觉重心**: 英雄区（大标题+视觉冲击背景）为第一视觉重心，排行榜区为功能重心
- **结果承载区**: 排行榜区承载用户提交后的AI分析结果和排名数据；初始态展示示例排行榜数据（标注为示例），或显示“暂无记录，快来上传你的挑战视频！”空状态

## 插件规划

| 插件实例名称 | 基于官方插件 | 业务用途 | 输出模式 | 所属页面 |
|------------|-----------|---------|---------|---------|
| 67速通视频分析 | `ai-video-understanding` | 分析用户上传的挑战视频，识别视频中打出了几个“67” | unary | 视频上传区 → 排行榜区 |

## 功能列表

- **区块: 英雄区** (`#hero`)
  - **区块目标**: 营造游戏速通挑战的震撼视觉冲击，传达核心玩法
  - **功能点**:
    - **大标题展示**: 超大字号白色粗体“67速通”，配合暗红色背景+闪电动效+红色熊形象，视觉张力极强
    - **标语展示**: “一分钟内打出尽可能多的67”，白色文字，位于标题下方
    - **CTA按钮**: “立即挑战”按钮，点击滚动到视频上传区（`#upload`）

- **区块: 视频上传区** (`#upload`)
  - **区块目标**: 让用户上传挑战视频并提交AI分析
  - **功能点**:
    - **视频上传**: 拖拽或点击上传视频文件，前端校验时长≤60秒、格式为常见视频格式（mp4/webm/mov），校验失败 toast 提示
    - **分区选择**: 下拉选择或按钮组选择分区（3个分区，如“键盘区”“手机区”“自由区”），必选
    - **用户名输入**: 文本输入框，输入挑战者昵称，必填
    - **视频简介输入**: 文本输入框（可选），简短描述挑战心得
    - **提交分析**: 点击“提交挑战”按钮，先校验表单完整性 → 显示上传进度/分析中状态 → 调AI视频分析插件 → 获取67数量 → 计算分数（67数量 ÷ 视频时长） → 写入localStorage排行榜 → toast“挑战成功！你的分数：X.XX” → 自动滚动到排行榜区

- **区块: 排行榜区** (`#leaderboard`)
  - **区块目标**: 按分区展示排名，营造竞技氛围
  - **功能点**:
    - **分区切换**: Tab切换3个分区，每个分区独立排名，切换时排行榜数据即时更新
    - **排名列表**: 表格或卡片列表展示，列含：排名（前三名特殊样式：金银铜）、用户名、分数（67数量÷视频时长，保留2位小数）、67数量、视频时长（秒）、提交时间
    - **空状态**: 当某分区无记录时，显示“暂无挑战记录，快来上传你的视频！”+ 跳转上传区链接
    - **新记录高亮**: 用户刚提交的记录在列表中短暂高亮闪烁（3秒），增强成就感

- **区块: 规则说明区** (`#rules`)
  - **区块目标**: 简单介绍玩法规则，降低理解门槛
  - **功能点**:
    - **规则展示**: 卡片式布局，列出核心规则：① 上传一分钟以内的视频 ② 视频内容为打出尽可能多的“67” ③ AI自动分析67数量 ④ 分数=67数量÷视频时长 ⑤ 分数越高排名越前 ⑥ 3个分区独立排名
    - **计分公式高亮**: 突出展示公式“分数 = 67的数量 ÷ 视频时长（秒）”，配合示例说明

# UI 设计指南

## 1. 设计推导依据

- **参考意图**: Exact Reference —— 参考图提供完整的视觉真相：暗红+亮黄撞色、粗体白字、霓虹发光数字、闪电特效、异形标签和搞怪生物形象，这些直接支配本产品的色彩、字体气质、装饰手法和 Hero 区布局。
- **核心情绪 / 应用类型**: 游戏速通挑战排行榜 —— 冲击力、速度感、竞技张力、趣味搞怪，类似短视频平台的趣味挑战宣传物料。
- **独特记忆点**: 暗红与亮黄的极端撞色 + 霓虹发光数字"67"作为品牌符号，闪电元素贯穿 Hero 与排行榜，营造"一分钟极限输入"的竞技紧张感。

## 2. Art Direction

- **方向名**: Neo Brutalism × Game Speedrun
- **Design Style**: Neo Brutalism 新野兽派 + 霓虹发光点缀 —— 粗边框、高对比撞色、粗体大字、异形标签和闪电装饰，完美匹配"67速通"的搞怪冲击力和游戏速通竞技感。
- **DNA 参数**: 圆角 sharp（`rounded-none` 或 `rounded-sm`，异形标签可用 `rounded-xl` 做局部反差）/ 阴影 heavy（`shadow-[4px_4px_0px]` 硬阴影，模拟参考图的黑色描边感）/ 间距 spacious（`gap-6`/`p-8`，Hero 区留足呼吸感）/ 字体方向 粗体无衬线 + 霓虹发光数字 / 装饰手法 闪电图标、霓虹光晕、粗黑描边、异形云朵标签。
- **应用类型**: Landing + Tool —— Hero 区全宽冲击，排行榜和上传区收束为工具型布局。

## 3. Color System

**色彩关系**: 暗红主色 + 亮黄辅助 + 暖金霓虹点缀 + 深黑文字与粗边框。
**配色设计理由**: primary 暗红来自参考图上半区背景，承载 Hero 区、主按钮和品牌识别；亮黄作为 accent 底和排行榜高亮行，复刻参考图下半区的撞色逻辑；暖金霓虹用于数字"67"和关键分数，强化速度感和竞技荣耀；深黑文字和粗边框保持 Neo Brutalism 的硬朗可读性。
**主色推导**: 参考图上半区为浓郁暖调深红，下半区为高饱和明黄，两者形成极端撞色；数字"67"为暖金色霓虹发光效果。primary 取暗红，accent 取亮黄，霓虹金作为语义高亮色。
**使用比例**: 55% 中性（深黑文字+浅灰卡片底）/ 30% 辅助（亮黄 accent 底+粗黑边框）/ 15% primary（暗红 Hero 区+主按钮+激活态）。

| 角色 | CSS 变量 | Tailwind Class | HSL 值 | 设计说明 |
|---|---|---|---|---|
| bg | `--background` | `bg-background` | hsl(50 30% 96%) | 暖白底，模拟参考图下半区的浅米色标签底色，柔和但不抢撞色 |
| card | `--card` | `bg-card` | hsl(0 0% 100%) | 纯白卡片，承载排行榜行和上传表单，与暗红 Hero 形成明暗对比 |
| text | `--foreground` | `text-foreground` | hsl(0 0% 8%) | 深黑正文，模拟参考图下半区黑色说明文字，高对比可读 |
| textMuted | `--muted-foreground` | `text-muted-foreground` | hsl(0 0% 35%) | 辅助说明、视频时长、分区标签等次级信息 |
| primary | `--primary` | `bg-primary` / `text-primary` | hsl(355 70% 38%) | 暗红主色，Hero 区背景、主按钮、排行榜第一名高亮，来自参考图上半区 |
| primaryForeground | `--primary-foreground` | `text-primary-foreground` | hsl(0 0% 98%) | 暗红底上的白色文字和图标，模拟参考图白色粗体标题 |
| accent | `--accent` | `bg-accent` | hsl(52 95% 55%) | 亮黄辅助，hover/focus 浅底、分区切换激活态、排行榜交替行，来自参考图下半区 |
| accentForeground | `--accent-foreground` | `text-accent-foreground` | hsl(0 0% 8%) | 亮黄底上的深黑文字，保持高对比 |
| border | `--border` | `border-border` | hsl(0 0% 8%) | 粗黑边框，Neo Brutalism 核心元素，模拟参考图绿色方块的黑边描边 |

**语义色提示**:
- **霓虹金（品牌符号色）**: 用于数字"67"、排行榜分数高亮、闪电图标发光效果。`hsl(42 100% 55%)`，搭配 `text-shadow: 0 0 12px hsl(42 100% 55% / 0.7)` 模拟霓虹光晕。此色不参与 primary/accent 体系，仅用于品牌符号和关键数据高亮。
- **成功色**: 上传成功、审核通过。bg `hsl(142 50% 92%)` / border `hsl(142 50% 45%)` / text `hsl(142 60% 25%)`，饱和度与 primary 对齐，不刺眼。
- **错误色**: 上传失败、格式不符。bg `hsl(0 50% 94%)` / border `hsl(0 55% 50%)` / text `hsl(0 60% 28%)`，色温与暗红 primary 同系但明度更高，区分功能。
- **警告色**: 视频时长超限。bg `hsl(42 60% 93%)` / border `hsl(42 70% 50%)` / text `hsl(42 65% 25%)`，与霓虹金同色温，保持系统一致性。

## 4. 字体与节奏

- **font-display**: ZCOOL QingKe HuangYou —— 圆润粗体中文，模拟参考图"67速通"的搞怪冲击力，适合 Hero 大标题和排行榜排名数字。
- **font-body**: Noto Sans SC —— 清晰可读的中文无衬线，承载上传表单、规则说明和排行榜用户名等正文信息。
- **字号**: H1 text-6xl ~ text-7xl（Hero 大标题，模拟参考图超大粗体字）；H2 text-3xl ~ text-4xl（分区标题）；body text-base；muted text-sm。
- **圆角**: sharp（`rounded-none` 或 `rounded-sm`）为主，异形标签和云朵按钮用 `rounded-xl` 做局部反差，模拟参考图的绿色方块（直角）与云朵标签（圆润）的对比。

## 5. 全局布局契约

- **Reference Layout Use**: Exact Reference —— Hero 区保留参考图上半区的全宽暗红背景+居中大标题+标语右排布局；排行榜和上传区采用参考图下半区的亮黄/浅米色分区逻辑，但改为工具型卡片布局以承载数据密度。
- **Page / Section Order**: Hero 区 → 视频上传区 → 排行榜区（含分区切换） → 规则说明区，与需求文档页面结构 1:1 对齐。
- **Standard Content Zone**: `max-w-5xl` + `mx-auto`，Hero 区文字和 CTA 受此约束，排行榜和上传表单在此宽度内保持可读行长。
- **Shell / Frame Alignment**: 内容容器与页面框架同宽，Hero 区背景全宽但内部内容区与下方卡片区对齐。
- **Padding & Rhythm**: `px-4 md:px-6 lg:px-8 py-12 md:py-16`，Hero 区上下留足呼吸感，排行榜区 `py-8`。
- **Full-bleed Zones**: Hero 区暗红背景 `w-full`，内部标题、标语和 CTA 按钮仍受 Standard Content Zone 约束；闪电装饰可突破内容区边界延伸到全宽。
- **Local Narrowing**: 上传表单在 `max-w-2xl` 内居中，规则说明区收窄至 `max-w-3xl` 提升阅读舒适度。
- **Overflow Strategy**: 排行榜表格在移动端使用 `overflow-x-auto`，不放大全局 max-w。
- **Flexibility Boundary**: 允许移动端 padding 和卡片内边距微调；不允许按页面切换 max-w、圆角系统、阴影语言或主色。

## 6. 视觉与动效

- **装饰**: 闪电图标（模拟参考图嘴部上方闪电特效）、霓虹发光数字"67"、粗黑描边异形标签、云朵轮廓按钮。
- **阴影/边界**: 重 —— `shadow-[4px_4px_0px_hsl(0_0%_8%)]` 硬阴影，模拟参考图绿色方块的黑色描边感，卡片和按钮统一使用。
- **动效**: 丰富 —— Hero 区标题入场使用 scale-in + 霓虹光晕脉冲；排行榜行 hover 时亮黄 accent 底滑入 + 硬阴影位移（`shadow-[6px_6px_0px]`）；数字"67"持续微弱的霓虹呼吸光晕；分区切换使用横向滑动过渡。

## 7. 组件原则

- 按钮、表单、菜单、卡片必须有 Default / Hover / Active / Focus / Disabled 状态，硬阴影在 hover 时位移加深，active 时回弹。
- Primary 按钮使用暗红底+白色粗体字+粗黑边框+硬阴影；Secondary/Outline 使用亮黄底+深黑文字+粗黑边框。
- 排行榜第一名行使用暗红 primary 底+白色文字+霓虹金分数高亮；其余行交替白底和亮黄 accent 浅底。
- 分区切换 Tab 使用 Neo Brutalism 风格：激活态暗红底白字+粗黑边框，非激活态白底黑字+粗黑边框。
- 上传按钮使用云朵轮廓异形样式（`rounded-xl` + 粗黑边框），模拟参考图的"(视频)"标签。
- 加载状态使用霓虹金脉冲 Skeleton，空状态使用搞怪生物插画+趣味文案，不回退到默认 shadcn 样式。

## 8. Image Direction

- **Image Role**: Hero 区背景主视觉 —— 替代参考图中的红毛熊形象，作为"67速通"的品牌吉祥物和视觉锚点。
- **Image Art Direction**: 一只造型夸张搞怪的拟人化红色生物（类似参考图的红毛熊），大张着嘴，嘴部上方迸发亮白色闪电特效，双爪捧着暖金色霓虹发光数字"67"。暗红渐变背景，生物主体居中偏左，右侧留给白色粗体大标题"67速通"和标语。光线从数字"67"的霓虹光晕向外辐射，生物毛发边缘有暖金 rim light。整体构图饱满、冲击力强，适合全宽 Hero 区，文字叠加区域保持暗色低细节以保证可读性。
- **Image Prompt Keywords**: 拟人化红色毛茸生物, 大张的嘴, 白色闪电特效, 暖金色霓虹发光数字67, 暗红渐变背景, 搞怪夸张表情, 游戏速通挑战, 霓虹光晕, rim lighting, 高对比度, 海报风格, 数字艺术
- **Image Avoidance**: 避免写实动物照片、可爱萌系卡通、通用科技感抽象背景、商务素材图库风、无主题渐变图；生物形象必须保持搞怪冲击力而非恐怖或血腥。

## 9. Anti-patterns

- **Split personality**: Hero 区 Neo Brutalism 重阴影撞色，排行榜区突然变成轻阴影圆角卡片；全站统一硬阴影语言和粗黑边框。
- **Phantom tokens**: 编造 `--neon-gold` 等未定义变量；霓虹金发光效果使用 `text-shadow` 内联样式或 Tailwind arbitrary value，不污染主题 token。
